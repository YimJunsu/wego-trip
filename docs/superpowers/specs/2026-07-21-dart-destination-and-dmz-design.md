# 다트 결과 → 여행지 동시 렌더링 + 휴전선 이북 착지 판정

작성일: 2026.07.21

## 배경

`/random` 의 다트 던지기는 시군구에 명중하면 지역명과 위경도만 보여준다. 정작
여행지(`mocks/destinations.json`)는 옆 탭 "조건으로 뽑기"에만 붙어 있어, 다트를
던져도 "그래서 어디 가라는 건데"가 안 나온다.

또 지도 데이터에는 남한 시군구 250개만 있어 휴전선 이북에 꽂히면 바다와 구분되지
않고 "바다에 빠졌습니다"가 뜬다.

## 목표

1. 다트가 명중하면 그 시도의 여행지 한 곳을 결과 카드 바로 아래에 함께 보여준다.
2. 휴전선 이북 착지를 바다와 구분해 "북한에는 갈 수 없습니다"로 안내한다.

## 비목표

- `mocks/destinations.json` 내용 보강 (여행지 0곳인 시도는 안내 문구로 처리)
- 지도 데이터에 북한 도형 추가, 북한 영역 시각화
- 공공데이터포털 관광 API 실연동 (인터페이스만 준비한다)

---

## 1. 다트 결과와 여행지 결과 동시 렌더링

### 현재 상태

- `DartGame` 의 결과는 `SigunguRegion` (250개, `code`/`name`/`province`/`centroid`)
- 여행지는 `Destination` (37개, `region` 이 `부산`·`강원`·`제주` 등 11개 축약 키)
- 두 데이터는 연결점이 없다

### 데이터 흐름

명중 → `region.province` (`강원특별자치도`) → 시도 키로 정규화 (`강원`) →
`destinationRepo.draw({ region })` → 다트 결과 카드 아래 `DestinationCard` 렌더.

### repository 계약 변경

```ts
// lib/data/types.ts
export type DestinationFilter = {
  /** 시도 단위 키('강원', '부산' …). PROVINCE_TO_REGION 이 생성한다. */
  region?: string
  themes?: DestinationTheme[]
  budget?: BudgetLevel
  season?: Season
}
```

지역 조회를 인터페이스에 넣어 둔다. 나중에 공공데이터포털 관광 API로 구현을 갈아끼울
때 화면과 계약을 손대지 않기 위해서다. mock 쪽은 `destinationRepo.match()` 에
`if (filter.region && dst.region !== filter.region) return false` 한 줄만 추가한다.

### province → region 매핑

`lib/utils/labels.ts` 에 상수를 추가한다.

```ts
/** 지도 데이터는 정식 명칭, 여행지 데이터는 축약 키를 쓴다. 둘을 잇는 유일한 표. */
export const PROVINCE_TO_REGION: Record<string, string> = {
  서울특별시: '서울',
  부산광역시: '부산',
  대구광역시: '대구',
  인천광역시: '인천',
  광주광역시: '광주',
  대전광역시: '대전',
  울산광역시: '울산',
  세종특별자치시: '세종',
  경기도: '경기',
  강원특별자치도: '강원',
  충청북도: '충북',
  충청남도: '충남',
  전북특별자치도: '전북',
  전라남도: '전남',
  경상북도: '경북',
  경상남도: '경남',
  제주특별자치도: '제주',
}
```

17개 시도를 모두 적는다. 표에 없는 값이 들어오면 여행지 조회를 건너뛰고 "여행지 없음"
안내로 처리한다 — 지도 데이터가 바뀌어도 화면이 깨지지 않는다.

### 여행지가 0곳인 시도

`destinations.json` 의 `region` 분포: 강원 7 · 전남 5 · 경북 5 · 경남 5 · 제주 4 ·
전북 3 · 부산 2 · 서울 2 · 충남 2 · 인천 1 · 충북 1.

**대구 · 광주 · 대전 · 울산 · 세종 · 경기** 6개 시도는 여행지가 없다. 이 경우
`EmptyState` 로 안내한다.

- 제목: "이 지역엔 아직 등록된 여행지가 없습니다"
- 설명: "다트는 잘 꽂혔어요. 여행방은 만들 수 있습니다."

다트 결과 카드와 "이 지역으로 여행방 만들기" 버튼은 그대로 뜬다.

### 로딩 처리

`destinationRepo.draw()` 는 비동기다(지금은 mock 150ms, 나중엔 실제 API). 다트 결과
카드는 기다리지 않고 즉시 뜨고, 여행지 영역만 `Skeleton` 으로 채운다.

눈 가리고 모드는 착지 후 450ms 뒤 `isRevealed` 가 켜지며 결과가 드러난다. 여행지
조회는 착지 시점에 시작하되 표시는 기존 reveal 타이밍을 따른다.

조회 실패 시 여행지 영역만 조용히 비운다. 다트 결과는 이미 나왔으므로 전체를
에러 화면으로 덮지 않는다.

### 상태 관리

`DartGame` 에 여행지 상태를 추가한다.

```ts
type DestinationState =
  | { status: 'idle' }
  | { status: 'pending' }
  | { status: 'found'; destination: Destination }
  | { status: 'none' }   // 해당 시도에 여행지 없음 또는 조회 실패
```

`land()` 에서 명중이면 조회를 시작한다. 다시 던지기(`reset`)에서 `idle` 로 되돌린다.
착지가 연달아 일어날 때 늦게 도착한 응답이 새 결과를 덮지 않도록, `RandomDrawer` 의
`useEffect` 가 쓰는 `isStale` 패턴과 같은 방식으로 무효화한다.

## 2. 휴전선 이북 착지 판정

### 원인

`lib/geo/korea-sigungu.json` 투영: `maxLat 38.6137` 이 `y=24`, `ky 200px/도`.
`DartGame.tsx` 의 착지 클램프가 `y ≥ 12` 까지 허용해 지도 상단 여백까지 꽂힌다.
`hitTest()` 가 시군구 path에 안 걸리면 무조건 `phase = 'miss'` → "바다에 빠졌습니다".
지도 데이터에 북한 도형이 없어 이북 전체가 바다로 처리된다.

### 새 파일 `lib/geo/dmz.ts`

```ts
/**
 * 군사분계선 근사 폴리라인. [경도, 위도], 경도 오름차순.
 * 정밀 국경이 아니라 "다트가 이북에 꽂혔나" 판정용 근사다.
 * 육지 판정이 먼저 돌기 때문에(§판정 순서) 이 선이 다소 어긋나도 남한 땅은 오판되지 않는다.
 */
const MDL: readonly [number, number][] = [
  [124.61, 37.72], // 서해 최서단 (NLL 근사)
  [126.24, 37.78], // 한강 하구 · 강화 북단
  [126.68, 37.95], // 판문점
  [127.03, 38.1], // 연천
  [127.3, 38.32], // 철원 북방
  [128.0, 38.32], // 양구 · 펀치볼
  [128.37, 38.62], // 고성 동해안
  [129.74, 38.62], // 동해 최동단
]

/** 경도로 구간을 찾아 위도를 선형 보간하고, 그보다 북쪽이면 true. */
export function isNorthOfMdl(lat: number, lng: number): boolean
```

폴리라인 양끝 바깥의 경도는 각각 첫 점·끝 점의 위도를 쓴다(외삽하지 않는다).

`lib/geo/` 에 별도 파일로 둔다. `koreaMap.ts` 는 지도 데이터 접근자이고 분계선은
그와 무관한 판정 규칙이라, 한 파일 한 책임 기준으로 섞지 않는다.

### 판정 순서 — 시군구 히트 → 북한 → 바다

이 순서가 설계의 핵심이다. 육지 판정이 먼저이므로 **남한 땅이 북한으로 오판될 수
없다.** 고성군·속초·양구처럼 38도선 위에 있는 남한 지역은 `hitTest()` 가 먼저 잡는다.
폴리라인은 "어느 시군구에도 안 걸린 점"에만 적용된다.

서해 NLL 이북 바다도 북한으로 본다. "바다인지 육지인지"보다 "저긴 못 간다"가 전달할
정보이므로 규칙을 하나로 유지한다.

### 코드 변경

- `DartGame` 의 `Phase` 에 `'north'` 추가
- `land()`: `hitTest` 실패 시 `unprojectPoint(target.x, target.y)` 로 좌표를 얻어
  `isNorthOfMdl()` 분기 → `'north'` 또는 `'miss'`
- `DartResultCard.tsx` 에 `DartNorthCard` 추가
  - 제목: "북한에는 갈 수 없습니다"
  - 설명: "아직은 국내 한정입니다. 휴전선 아래로 다시 던지세요."
  - `DartMissCard` 와 같은 구조·톤, 다시 던지기 버튼 동일
- 눈 가리고 모드는 착지를 남한 시군구 대표점(`anchor`)으로 스냅하므로 이 분기를 타지
  않는다. 의도된 동작이라 그대로 둔다.
