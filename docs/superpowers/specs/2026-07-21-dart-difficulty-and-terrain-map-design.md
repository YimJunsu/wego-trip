# 다트 난이도 상향 + 지도 지형 채색

작성일: 2026.07.21

## 배경

**다트가 너무 쉽다.** 조준선이 착지점을 그대로 보여주고 무작위 요소가 하나도 없어,
몇 번 던져 보면 원하는 지역에 정확히 꽂을 수 있다. "랜덤 여행지 뽑기"인데 랜덤이
아니다.

**지도가 회색이다.** 시군구를 먹색 명도 4단계로만 칠해 한반도 형태는 살지만 지형이
읽히지 않는다.

## 목표

1. 다트 착지에 진짜 무작위성을 넣어 결과를 예측할 수 없게 한다.
2. 지도를 지형이 읽히는 자연색으로 칠한다 — 바다·평야·구릉·산지.

## 비목표

- 다트 물리를 실제 탄도학으로 바꾸기 (손맛용 근사면 충분하다)
- 정밀 고도 데이터(DEM) 도입
- 지도 데이터(`korea-sigungu.json`) 재생성
- 눈 가리고 모드 동작 변경

## 1. 다트 난이도

### 원인

```
DartGame.tsx:262-273  조준선을 landingPoint(HOME, pull)까지 긋는다 — 착지 예정지 그 자체
DartGame.tsx:233-237  land()가 같은 좌표를 그대로 쓴다
lib/geo/dart.ts       landingPoint는 순수 결정론. 난수 없음
```

당긴 만큼 정확히 그 자리에 꽂힌다.

### 무작위 두 겹

| | 성격 | 크기 |
| --- | --- | --- |
| **바람** | 던지기 전에 **보여준다**. 학습·보정 가능 | 방향 무작위, 세기 0.3~1.0 → 최대 약 110px 편류 |
| **산포** | **숨긴다**. 파워에 비례하는 손떨림 | `90px × power` 반경 원 안 균등 분포 |

둘을 같이 넣는 것이 핵심이다. 바람만 있으면 보이니까 결국 다시 계산 가능해지고,
산포만 있으면 왜 빗나갔는지 몰라 억울하다. **바람은 납득을, 산포는 진짜 불확실성을**
담당한다.

시군구 평균 크기는 약 58px(`√(855×1000/250)`)이다. 산포 90px이면 풀파워에서 옆 동네로
새는 것이 정상이 된다. 살살 던지면 정확해지지만 멀리 못 간다 — 파워와 정확도가
맞바꿔진다.

### 조준선

착지점까지 긋지 않는다. HOME에서 조준 방향으로 **짧은 고정 길이**(약 150px)만 남긴다.
어디로 갈지가 아니라 어느 쪽으로 갈지만 알려준다.

### 바람을 굴리는 시점

`reset()`과 최초 마운트. 조준하는 동안은 고정이다. 던지는 순간 굴리면 보정할 방법이
없어 그냥 불공평해진다.

바람은 지도 위에 방향 화살표와 세기로 표시한다.

### 비행 연출

다트가 바람 방향으로 휘어 날아간다. 시작·끝을 고정하고 중간만 부풀린다.

```
pos = lerp(HOME, landing, k) + bow × (k − k²) × 4
```

`k − k²`는 양끝에서 0, `k=0.5`에서 최대 0.25다. 4를 곱해 중간 최대 편차가 `bow`가
되게 한다. 착지점은 계산값 그대로 유지된다.

### 순수 함수

계산은 전부 `lib/geo/dart.ts`에 둔다. 난수를 주입받게 만들어 검증 스크립트로 분포를
측정할 수 있게 한다 (CONVENTIONS §3 "부수효과 없는 계산은 순수 함수로").

```ts
export type Wind = { angle: number; strength: number }

/** 매 라운드 굴린다. strength는 0.3~1.0. */
export function rollWind(rand?: () => number): Wind

/** 파워에 비례하는 원판 균등 분포. √u를 써야 가장자리로 쏠리지 않는다. */
export function scatterOffset(power: number, rand?: () => number): Vec

/** 조준점 + 산포 + 바람 편류. 최종 착지점. */
export function resolveLanding(
  home: Vec,
  pull: Vec,
  wind: Wind,
  rand?: () => number,
): Vec
```

상수는 `MAX_SCATTER = 90`, `MAX_WIND_DRIFT = 110`, `AIM_LINE_LENGTH = 150`.

### 눈 가리고 모드

착지를 무작위 시군구 대표점으로 스냅하므로 바람·산포가 의미 없다. 그대로 둔다.

### 기존 동작 보존

`prefers-reduced-motion`이면 애니메이션 없이 바로 착지한다. 이때도 최종 착지점은
같은 `resolveLanding` 결과여야 한다 — 모션 설정에 따라 난이도가 달라지면 안 된다.

## 2. 지도 지형 채색

### 제약

`korea-sigungu.json`의 region 필드는 `code`·`name`·`province`·`path`·`centroid`·
`anchor` 뿐이다. **고도 데이터가 없다.** 출처가 통계청 2018 시군구 경계 TopoJSON이라
행정구역 폴리곤만 들어 있다.

### 고도 등급 — 손으로 지정한 근사

`lib/geo/terrain.ts`에 시군구 코드 → `'low' | 'mid' | 'high'` 맵을 둔다. 명시하지
않은 곳은 `low`.

- `high` — 태백산맥(평창·정선·인제·양구·태백·삼척 등), 소백산맥(단양·영월·봉화·
  문경·무주·장수), 지리산권(함양·거창·산청·하동)
- `mid` — 그 주변 시군구
- `low` — 나머지 (서해안 평야, 도시)

결과적으로 동쪽이 높고 서쪽이 낮은 경동성 지형이 드러나 한반도답게 보인다.

**정밀 고도가 아니라 일러스트용 근사다.** 파일 주석에 명시한다. 나중에 실제 DEM을
붙이면 이 파일만 교체하면 된다 — 소비하는 쪽(`KoreaMapLayer`)은 등급만 알면 된다.

### 팔레트

`app/globals.css`의 `@theme`에 추가한다. 채도를 낮춘 자연색이라 라임과 싸우지 않고
미색 배경(`#F4F4EF`)과 붙는다.

```
--color-terrain-sea       옅은 청록 회색   바다
--color-terrain-sea-deep  한 단계 짙게     먼바다 (지도 가장자리)
--color-terrain-low       연한 미색 녹     평야
--color-terrain-mid       세이지           구릉
--color-terrain-high      짙은 올리브      산지
```

정확한 hex는 구현 시 실제 화면을 보고 정한다. 색은 실물로 봐야 판단이 된다.

### 렌더링

`KoreaMapLayer` 안에서:

1. 맨 뒤에 바다 `rect` 한 장 (`terrain-sea`)
2. 그 위에 시군구 `path`를 등급별 색으로
3. 경계선은 흰 실선 (지금과 동일)
4. **명중 지역은 지금처럼 `fill-lime`** — 자연색 위에서 오히려 더 튄다

`PROVINCE_TINT`(시도별 먹색 명도 배정)는 지형 등급이 대체하므로 제거한다.

### 디자인 시스템 예외

`DESIGN_SYSTEM.md` §1은 "강조색은 라임 하나. 원색을 여러 개 쓰지 않는다"고 못박고
있고, `KoreaMapLayer.tsx` 주석도 이를 근거로 먹색만 쓴다고 적혀 있다.

"지도 팔레트(일러스트 예외)" 절을 신설한다.

- 범위를 **지도 SVG 안으로 한정**한다
- 버튼·카드·배지 등 UI 크롬은 여전히 라임 하나
- 지도는 UI가 아니라 콘텐츠(일러스트)라서 자체 팔레트를 갖는다는 근거를 적는다
- `KoreaMapLayer.tsx`의 "색은 늘리지 않고" 주석도 함께 고친다

## 검증

1. `pnpm lint` · `pnpm typecheck` · `pnpm build`
2. **예측 불가 증명** — 브라우저에서 똑같은 pull로 20회 던져 착지 좌표를 수집한다.
   전부 다른 지점이고 산포가 설계값 근처인지 실측해 보고한다. 지금은 20회 모두 같은
   좌표가 나오므로 그대로 before/after 대비가 된다.
3. **지형 데이터 무결성** — `terrain.ts`의 모든 코드가 `korea-sigungu.json`에
   실재하는지 스크립트로 대조한다 (오타·유령 코드 0건). 등급별 시군구 수도 함께 센다.
4. **채색 육안** — 실제 지도를 스크린샷해서 보고한다. 목업을 만들지 않는다.
5. **회귀 확인** — 명중/바다/북한 판정, 눈 가리고 모드, 여행지 연동이 그대로인지
   브라우저에서 재확인한다.

색감이 마음에 안 들면 토큰 5개만 고치면 된다.

## 리스크

- **난이도가 과해질 수 있다** → 상수 3개(`MAX_SCATTER`·`MAX_WIND_DRIFT`·
  `AIM_LINE_LENGTH`)로 모아 두어 숫자만 고치면 조절된다. 실측 분포를 보고한 뒤
  같이 조정한다.
- **고도 근사가 틀린 곳이 있다** → 일러스트 목적이라 치명적이지 않다. 등급 맵만
  고치면 되고, 지도 데이터나 렌더링은 건드릴 필요가 없다.
- **DartGame이 더 커진다** → 이미 449줄로 컨벤션 200줄을 넘겼다. 계산은 전부
  `lib/geo/dart.ts` 순수 함수로 빼서 컴포넌트에는 상태와 표현만 남긴다. 분해는
  `2026-07-21-mock-ui-removal-and-refactor-design.md` §5에서 다룬다.
